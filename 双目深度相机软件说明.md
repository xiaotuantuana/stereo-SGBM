# 纯视觉双目深度相机软件说明

## 启动方式

双击：

```text
启动双目深度相机软件.bat
```

或在当前目录运行：

```bash
python stereo_depth_camera_app.py
```

## 已实现功能

1. 支持左右拼接双目视频输入，默认自动寻找项目内可用视频，优先打开根目录下的 `car.avi`。
2. 支持相机输入，可从系统可打开的相机接口中识别选择，并可填写采集分辨率和相机帧率；只识别到 1 个相机时只显示左图，识别到 2 个相机时使用前两个接口作为左右相机做双目测距。
3. 支持 BM / SGBM 两种传统立体匹配方法。
4. 支持立体校正，内置原仓库 `640x480` 和 `1280x720` 两套标定参数，也可以在右侧界面手动填写标定参数。
5. 支持 WLS、中值滤波、小孔洞填充、双边边缘保持后处理。
6. 实时显示左图、右图、视差图、深度图、FPS 和视差有效率，左右图右上角叠加 FPS、右下角叠加时间。
7. 支持启动前“预览选点”：视频可填写预览起始帧并播放预览，到目标画面后点击“暂停选点”再选择测点。
8. 可保存截图、视差矩阵、深度矩阵和当前参数。
9. 可导出逐帧测距 CSV，每一帧都会保存每个确认测点的距离、三维坐标、视差和耗时。
10. 右侧参数面板支持鼠标滚轮上下滚动，并已按两列组织可选/可填参数。
11. 图像区和参数区之间支持拖动分栏，左/右两列图像之间也支持拖动调宽；初始状态下四个图像窗口会尽量均分显示。

## 手动标定参数

右侧“相机标定参数”中默认使用“内置匹配”。如果需要使用自己的相机参数：

1. 将“标定来源”改成“手动填写”。
2. 填写单目宽高，必须等于输入画面左右拆分后的单目尺寸。
3. 填写左右相机的 `fx/fy/cx/cy`、左右畸变 `k1,k2,p1,p2,k3`、旋转矩阵 `R` 的 9 个数、平移向量 `T` 的 3 个数。
4. 平移向量通常使用标定结果的毫米单位；程序会沿用该单位并在深度显示时自动换算为米。

## 推荐默认参数

当前默认值来自前期模拟实验中较稳定的改进 SGBM：

| 参数 | 默认值 |
|---|---:|
| method | SGBM |
| blockSize | 3 |
| numDisparities | 64 |
| uniquenessRatio | 15 |
| speckleWindowSize | 150 |
| speckleRange | 2 |
| disp12MaxDiff | 1 |
| 小孔洞填充 | 开 |
| 双边保边 | 开 |
| WLS | 默认关 |

## 输出目录

软件截图和测距 CSV 会保存到：

```text
stereo_app_outputs/
```

每次截图会生成一个时间戳文件夹，包含：

- `left.png`
- `right.png`
- `disparity_color.png`
- `depth_color.png`
- `disparity.npy`
- `depth_m.npy`
- `settings.json`
- `points.json`

## 测点使用方式

1. 启动视频或相机。
2. 在右下角深度图上单击目标位置。
3. 底部会显示“待确认”的 X/Y/Z、距离和视差。
4. 深度图上会显示 `P? 距离m` 的待确认标记，不再显示中文问号。
5. 点击顶部“确定测点”，测点会以十字、编号和距离显示在深度图上，例如 `P1 6.24m`。
6. 如果点错，点击“清空测点”重新选择。
7. 点击“导出逐帧CSV”保存播放过程中每一帧、每个测点的数据。

## 推荐逐帧测距流程

1. 选择视频文件或双目相机。
2. 如果是视频，可填写“预览起始帧”，点击“预览选点”后播放到合适画面。
3. 点击“暂停选点”，在深度图上点击目标点，底部显示待确认距离。
4. 点击“确定测点”，可重复添加 P1、P2、P3 等多个测点。
5. 点击“启动”，软件从第 1 帧开始播放，并对所有确认测点逐帧采样。
6. 停止后点击“导出逐帧CSV”。

逐帧 CSV 字段包括：

- `frame`
- `time`
- `point_id`
- `x`
- `y`
- `valid`
- `X_m`
- `Y_m`
- `Z_m`
- `distance_m`
- `disparity`
- `fps`
- `elapsed_ms`

## 注意事项

1. 如果你的双目相机不是左右拼接输出，需要先改采集逻辑。
2. 如果相机分辨率不是 `1280x720` 或 `640x480` 单目尺寸，需要重新标定，并在界面中选择“手动填写”录入参数。
3. WLS 不一定总能提升误差指标，当前默认关闭；如果画面噪声多，可以手动打开调试。
4. “识别相机”只能列出 OpenCV 能打开的相机接口；如果只识别到 1 个相机，软件不会把单目画面强行切成左右图。
5. 如果厂商相机需要专用 SDK，需要再接入对应 SDK。
6. 点击测距依赖标定参数和视差质量，标定不准时距离会明显漂移。
7. 这个软件优先实现“纯视觉双目深度相机”的可用界面和传统算法流程，后续可继续加入 YOLO 目标测距、参数预设管理、相机标定向导和三维点云显示。
