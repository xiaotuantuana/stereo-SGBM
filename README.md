# 纯视觉双目深度相机软件

一个基于 Python、OpenCV 和 Tkinter 的双目视频 / 双目相机测距软件。软件面向左右拼接 AVI 双目视频和常见 USB 相机接口，提供实时左图、右图、视差图、深度图显示，支持手动填写相机标定参数、点击选点测距、截图保存和逐帧 CSV 导出。

## 项目特点

- 支持左右拼接双目视频输入，常见格式包括 `.avi`、`.mp4`、`.mov`、`.mkv`。
- 支持系统相机接口识别，只识别到 1 个相机时只显示左图，不会强行切分画面；识别到 2 个相机时使用前两个接口作为左右相机进行双目测距。
- 支持 BM / SGBM 两种传统立体匹配方法。
- 支持内置标定参数，也支持在界面手动填写左右相机内参、畸变、旋转矩阵和平移向量。
- 支持 WLS、中值滤波、小孔洞填充、双边滤波等后处理选项。
- 支持视频预览选点：可填写起始帧，播放到目标画面后暂停并选择测点。
- 支持保存截图、深度矩阵、视差矩阵、当前参数和测点数据。
- 支持导出逐帧测距 CSV。

## 运行环境

推荐环境：

- Windows 10 / Windows 11
- Python 3.10 到 3.12
- 普通 USB 摄像头、双目相机，或左右拼接双目视频

Python 依赖：

```text
numpy
pillow
opencv-contrib-python
```

`opencv-contrib-python` 包含 WLS 相关的 `cv2.ximgproc` 模块。如果不使用 WLS，也可以改用普通 `opencv-python`，但推荐保持默认依赖。

## 快速开始

### 方法一：Windows 双击启动

下载仓库后，双击：

```text
启动双目深度相机软件.bat
```

首次运行会自动：

1. 创建 `.venv` 虚拟环境。
2. 安装 `requirements.txt` 中的依赖。
3. 启动软件界面。

首次安装依赖需要联网。后续再次启动会直接使用已安装好的本地虚拟环境。

### 方法二：命令行运行

```bash
pip install -r requirements.txt
python stereo_depth_camera_app.py
```

## 使用流程

### 使用双目视频

1. 在左上角选择“视频”。
2. 点击“选择视频”，选择左右拼接的双目视频文件。
3. 如果需要从指定位置开始预览，填写“预览起始帧”。
4. 点击“预览选点”，视频会开始播放。
5. 到达目标画面后点击“暂停选点”。
6. 在深度图上点击目标位置，再点击“确定测点”。
7. 点击“启动”开始正式播放并逐帧记录测距结果。
8. 点击“导出逐帧CSV”保存数据。

视频要求：左右画面应水平拼接在同一帧中，例如总宽度为 `1280`，则左图和右图各为 `640` 宽。

### 使用相机

1. 在左上角选择“双目相机”。
2. 点击“识别相机”。
3. 如果只识别到 1 个相机，软件只显示左图，不进行双目测距。
4. 如果识别到 2 个相机，软件默认使用前两个相机作为左相机和右相机。
5. 根据相机实际输出填写采集分辨率和帧率模式。
6. 点击“预览选点”或“启动”。

注意：相机识别基于 OpenCV 可打开的系统相机接口。如果你的工业相机、厂商双目相机需要专用 SDK，需要在代码中接入对应 SDK 后才能使用。

## 标定参数

软件右侧“相机标定参数”区域支持两种模式：

- `内置匹配`：使用程序内置的 `640x480` 和 `1280x720` 标定参数。
- `手动填写`：使用界面中填写的标定参数。

手动填写时需要提供：

- 单目宽、高
- 左相机 `fx/fy/cx/cy`
- 右相机 `fx/fy/cx/cy`
- 左右相机畸变参数 `k1,k2,p1,p2,k3`
- 旋转矩阵 `R` 的 9 个数
- 平移向量 `T` 的 3 个数

单目尺寸必须和输入图像拆分后的左右图尺寸一致，否则会提示尺寸不匹配。平移向量通常使用标定结果中的毫米单位，软件会在深度显示时换算为米。

## 界面功能

- `启动`：打开当前输入源并开始实时处理。
- `停止`：停止当前播放或采集。
- `预览选点`：进入预览播放，用于找到合适帧。
- `暂停选点`：冻结当前预览帧，便于点击测点。
- `确定测点`：保存当前点击位置为测点。
- `清空测点`：清空所有测点和逐帧记录。
- `截图/保存`：保存当前左图、右图、视差图、深度图、矩阵和参数。
- `导出逐帧CSV`：导出已确认测点的逐帧测距结果。

## 输出文件

截图和 CSV 默认保存到：

```text
stereo_app_outputs/
```

每次截图会创建一个时间戳目录，包含：

- `left.png`
- `right.png`
- `disparity_color.png`
- `depth_color.png`
- `disparity.npy`
- `depth_m.npy`
- `settings.json`
- `points.json`

逐帧 CSV 字段包括：

- `frame`
- `time`
- `point_id`
- `x`
- `y`
- `valid`
- `X_m`
- `Y_m`
- `Z_m`
- `distance_m`
- `disparity`
- `fps`
- `elapsed_ms`

## 常见问题

### 选择 AVI 后打不开

请确认：

- 文件路径真实存在。
- AVI 文件没有损坏。
- 当前 OpenCV 支持该 AVI 的编码格式。
- 路径中包含中文时，建议使用软件内的“选择视频”按钮选择文件。

### 只有一个相机时为什么没有深度图

双目测距需要左、右两个不同视角。只识别到一个相机时，软件只做单目显示，不会把同一个画面切成左右两半来制造错误深度。

### 深度结果不准

常见原因：

- 标定参数不匹配当前相机。
- 单目尺寸和标定尺寸不一致。
- 左右图不同步。
- 双目相机基线或内参填写错误。
- 目标区域纹理太弱，视差质量低。

### WLS 无法使用

请确认安装的是：

```bash
pip install opencv-contrib-python
```

而不是普通的 `opencv-python`。

## 已知限制

- 当前相机接入基于 OpenCV `VideoCapture`，不包含厂商 SDK。
- 当前默认假设双目视频为水平左右拼接格式。
- 单相机模式只用于预览显示，不提供深度测距。
- 点云显示、YOLO 目标检测测距和自动标定向导暂未内置。
